# 模式: ying-zheng -- 應徵線上助手

候選人正在瀏覽器填寫應徵表單時的互動模式。讀取螢幕上的內容，載入事先評估的上下文，為表單的每個問題生成個人化回答。

## 前置條件

- **有 Playwright 效果最好**：候選人開著瀏覽器，Claude 可以與頁面互動。
- **沒有 Playwright**：候選人分享截圖或手動貼上問題。

## 工作流程

```
1. 偵測    → 讀取 Chrome 當前分頁（截圖/URL/標題）
2. 識別    → 從頁面擷取公司名 + 職位
3. 搜尋    → 比對 reports/ 中的現有報告
4. 載入    → 讀取完整報告 + 區塊 H（若存在）
5. 比較    → 螢幕上的職缺是否與已評估的一致？有變更就通知
6. 分析    → 識別表單上所有問題
7. 生成    → 為每個問題生成個人化回答
8. 呈現    → 格式化顯示可複製貼上的回答
```

## 步驟 1 -- 偵測職缺

**有 Playwright：** 取得當前頁面快照。讀取標題、URL 和顯示內容。

**沒有 Playwright：** 請候選人：
- 分享表單截圖（Read tool 可讀取圖片）
- 或將表單問題貼成文字
- 或告訴公司名 + 職位，用來搜尋上下文

## 步驟 2 -- 識別並搜尋上下文

1. 從頁面擷取公司名和職位名稱
2. 用公司名搜尋 `reports/`（Grep，不區分大小寫）
3. 有匹配 → 載入完整報告
4. 有區塊 H → 以之前的回答草稿為基礎
5. 無匹配 → 告知候選人，建議先執行快速 auto-pipeline

## 步驟 3 -- 偵測職缺變動

若螢幕上的職缺與已評估的不同：
- **告知候選人：**「職缺已從 [X] 改為 [Y]。要重新評估，還是直接調整回答？」
- **調整的話：** 不重新評估，直接調整回答到新職位
- **重新評估的話：** 執行完整評估 A-F，更新報告，重新生成區塊 H
- **更新 tracker：** 必要時修改 `applications.md` 中的職位名稱

## 步驟 4 -- 分析表單問題

識別所有顯示的問題：
- 自由文字欄（自我介紹、應徵動機、為何選擇此公司等）
- 下拉選單（如何得知此職缺、工作資格等）
- 是/否問題（可否搬遷、需不需要簽證、可到職時間等）
- 薪資欄（期望薪資、目前薪資）
- 上傳欄（履歷、cover letter PDF）

將每個問題分類：
- **區塊 H 已回答過** → 調整既有回答
- **新問題** → 從報告 + `cv.md` 生成回答

## 步驟 5 -- 生成回答

依照以下原則為每個問題生成回答：

1. **報告上下文**：使用區塊 B 的 proof points、區塊 F 的 STAR 故事
2. **之前的區塊 H**：有草稿的話以它為基礎，加以精煉
3. **「你是我選擇的公司」語氣**：自信，不卑微
4. **具體性**：引用螢幕上 JD 的具體內容
5. **career-ops 的 proof points**：若有「補充說明」欄位就放進去

**台灣求職表單常見特有問題：**
- **期望薪資（月薪/年薪，含稅）** → 用 TWD 填寫 `profile.yml` 的範圍，標注「可依整體 package 討論」
- **目前薪資（含稅）** → 依 `profile.yml` 填寫；若不想揭露，可填「面議」
- **可到職日** → 合理日期，考量目前公司的離職預告期（通常 1 個月）
- **工作資格（就業證/居留證）** → 清楚回答；台灣人：「中華民國國籍，無需工作許可」
- **語言能力** → 依語言列出程度（母語、商業英文、日常溝通等）
- **可否配合出差/外派** → 依 `profile.yml` 的地點設定回答
- **離職原因（目前/前份工作）** → 正面框架，聚焦成長目標，不批評前雇主
- **家庭狀況（注意：此問題屬個人隱私，候選人有權拒絕回答）** → 告知候選人有權不回答此問題

**輸出格式：**

```
## [公司名] -- [職位名稱] 回答

基礎: 報告 #NNN | 分數: X.X/5 | 職缺類型: [類型]

---

### 1. [表單問題（原文）]
> [可複製貼上的回答]

### 2. [下一個問題]
> [回答]

...

---

備注:
- [對此職缺的觀察、變動等]
- [建議候選人確認的個人化部分]
```

## 步驟 6 -- 應徵後（選填）

候選人確認已送出應徵後：
1. 將 `applications.md` 中的狀態從「Evaluated」更新為「Applied」
2. 以最終回答更新報告的區塊 H
3. 建議下一步：LinkedIn 外聯用 `/career-ops contacto`

## 滾動處理

若表單問題超出可視範圍：
- 請候選人滾動並分享另一張截圖
- 或請他們貼上剩餘問題
- 重複直到覆蓋整個表單
